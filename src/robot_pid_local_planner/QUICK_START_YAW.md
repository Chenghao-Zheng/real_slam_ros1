## 快速使用指南 - 终点朝向控制

### 一句话说明
小车在到达导航目标点后，会自动旋转到你在 RViz 中用 2D Nav Goal 指定的朝向。

---

## 三步启用

### 1️⃣ 修改参数
编辑 `src/robot_pid_local_planner/config/omnidirectional_pid_param.yaml`：
```yaml
ENABLE_FINAL_YAW: true    # 启用终点朝向约束
YAW_TOL: 0.05             # 朝向容差（弧度）
```

### 2️⃣ 重新编译
```bash
cd ~/RealSlamRos1
catkin_make --pkg robot_pid_local_planner -j4
```

### 3️⃣ 使用
- 启动小车：`bash real_lidar_slam.sh`
- 在 RViz 中点击 **"2D Nav Goal"** 按钮
- **点击位置**，然后**拖动鼠标**设置朝向
- 小车会先走到位置，再转向到指定朝向

---

## 参数说明

| 参数 | 含义 | 默认值 | 建议范围 |
|------|------|--------|---------|
| `ENABLE_FINAL_YAW` | 启用终点朝向控制 | `true` | - |
| `YAW_TOL` | 朝向容差（弧度） | `0.05` | 0.03~0.15 |
| `KP_YAW` | 朝向 P 增益 | `1.2` | 0.5~2.0 |

**容差参考**：
- 0.03 rad ≈ 1.7°（很严格）
- 0.05 rad ≈ 2.9°（推荐）
- 0.10 rad ≈ 5.7°（较宽松）
- 0.20 rad ≈ 11.5°（很宽松）

---

## 工作流程

```
小车运动
  ↓
到达目标位置？
  ├─ 否 → 继续前进
  └─ 是 → 进入朝向调整
       ↓
    启用朝向约束？
      ├─ 否 → 停止（完成）
      └─ 是 → 调整朝向
           ↓
        朝向满足？
          ├─ 否 → 继续旋转
          └─ 是 → 停止（完成）
```

## 故障排查

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 小车到达后不旋转 | 参数未加载或编译过期 | 重新编译 + 检查参数 |
| 旋转速度很慢 | `KP_YAW` 过小 | 增大 `KP_YAW`（如 2.0） |
| 旋转很抖 | `KP_YAW` 过大或 `YAW_TOL` 过严 | 降低 `KP_YAW` 或增大 `YAW_TOL` |
| RViz 没有朝向箭头 | 2D Nav Goal 用法不对 | 在地图上**拖动鼠标**（不是只点击） |

---

## 禁用功能

如果想恢复原来的行为（到达位置后立即停止，不考虑朝向）：
```yaml
ENABLE_FINAL_YAW: false
```

重新编译后，小车会在到达位置后立即停止。
